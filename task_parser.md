# Задача: Создание модуля парсинга Google Maps (Playwright) + AI Анализ

## Контекст
У нас есть готовый бэкенд FastAPI для AI SDR проекта. Нужно добавить новый функционал: асинхронный парсер Google Maps на базе Playwright, который будет искать компании, собирать их данные, анализировать их "боли" через Anthropic API и сохранять как новых лидов.

## Требования к разработке:
1. **Установка зависимостей:** Добавь `playwright` и `beautifulsoup4` в requirements.txt (или pyproject.toml).
2. **Endpoint:** Создай новый эндпоинт `POST /api/parser/run`. 
   - Тело запроса (Pydantic модель): `location` (строка, например "Киев"), `keyword` (строка, например "стоматология"), `limit` (число, например 10).
3. **Фоновая задача (BackgroundTasks):** Парсинг — долгий процесс. Эндпоинт должен сразу возвращать `{"status": "started"}`, а сам парсинг запускать в фоне.
4. **Логика Playwright (Скрипт):**
   - Открыть `https://www.google.com/maps`.
   - Вбить в поиск "{keyword} {location}".
   - Дождаться загрузки боковой панели с результатами.
   - Собрать данные первых `limit` компаний: Название, Сайт (если есть), Рейтинг.
5. **AI Анализ (Боль клиента):**
   - Для каждой найденной компании сформировать микро-промпт для Claude 4.6: "Компания: [Название], Ниша: [Keyword], Рейтинг: [Рейтинг]. Придумай 1 реалистичную проблему (боль) для этой компании, которую мы могли бы решить нашей услугой автоматизации. Ответь одним коротким предложением."
6. **Сохранение:** - Найти email на сайте компании (можно сделать простую функцию поиска `mailto:` через httpx + BeautifulSoup по сайту компании).
   - Сохранить лида в БД (таблица `leads`) со статусом `new`, источником `parser` и заполненным полем `pain_point`.

Напиши весь необходимый код. Раздели логику на `routers/parser.py` и `services/scraper_service.py`.
